# team/ — 페르소나별 작업 산출물

각 서브에이전트의 개인 폴더 구조:

```
team/
├── team-lead/          팀장 — 업무 분배 계획, 취합 보고서
├── planner/            기획자 — 기획서, 요구사항, 플로우차트
├── frontend-dev/       프론트엔드 — UI 구현 기록, 컴포넌트 가이드
├── backend-dev/        백엔드 — API 설계서, 아키텍처 결정
├── db-architect/       DB 아키텍트 — ERD, 쿼리 튜닝, 마이그레이션 계획
├── ui-designer/        디자이너 — 디자인 가이드, 목업, 스타일 정의
├── devops-engineer/    DevOps — 배포 설정, 파이프라인, 인프라 문서
├── security-expert/    보안 — 보안 감사, 취약점 분석, 정책 가이드
├── qa-tester/          QA — 테스트 결과 보고서
└── researcher/         리서처 — 기술 조사 보고서, 비교 분석
```

## 파일명 규칙

`YYYY-MM-DD_{작업주제}.md`

예: `2026-03-06_사용자인증API설계.md`
