# 작업 내용 정리 (2026-03-27)

## 목표
application.yml의 `edgesquare.mode` 하드코딩을 제거하고, 라이선스 파일(`license_EdgeSquare`)의 `sLicenseTrgtCd` 값으로 제품 모드(monitor/manager)를 자동 결정하도록 변경

## 완료된 작업

### 1. LicenseEnvironmentPostProcessor 신규 생성
- **파일**: `common/config/LicenseEnvironmentPostProcessor.java`
- **역할**: Spring 컨텍스트 생성 **전**에 라이선스 파일을 읽어 `edgesquare.mode` 프로퍼티를 동적 설정
- **모드 판별 로직**: `sLicenseTrgtCd` 값(예: `"32,33"`)에서 코드 `33`이 포함되면 `monitor`, 없으면 `manager`
  - 32 = manager, 33 = monitor
- **폴백**: 라이선스 파일 없음 / 코드 `"0"` / 복호화 실패 → 기본값 `monitor`
- **등록**: `META-INF/spring.factories`에 EnvironmentPostProcessor로 등록

### 2. application.yml edgesquare.mode 제거
- 기존 `edgesquare.mode: monitor` 하드코딩을 주석 처리
- 라이선스 기반으로 자동 결정됨을 주석으로 설명 추가

### 3. LicenseService → em_preference DB 동기화
- **파일**: `common/service/LicenseService.java`
- `postConstruct()`에서 라이선스 로드 후 `syncEdgesquareModeToPreference()` 호출
- `edgesquare.mode` 키가 em_preference에 없으면 INSERT, 있으면 라이선스 값과 다를 때 UPDATE
- 프론트엔드 `cm.getSystemProperty("edgesquare.mode")`가 DB에서 값을 읽으므로 동기화 필수

### 4. ContentFilesRepository Oracle 호환 수정
- **파일**: `common/repository/ContentFilesRepository.java`
- `findByFileTypeIsStartingWith()` → `@Query` JPQL로 변경
- Hibernate 자동 생성 `escape '\\'`가 Oracle에서 `ORA-01425` 에러 발생 → `CONCAT(:fileType, '%')` 방식으로 해결

## 동작 흐름

```
Spring 기동
  ↓
① LicenseEnvironmentPostProcessor (컨텍스트 생성 전)
   license_EdgeSquare 파일 → DES 복호화 → sLicenseTrgtCd 추출
   33 포함? → edgesquare.mode = "monitor" / 미포함 → "manager"
  ↓
② Spring 프로퍼티로 주입
   @ConditionalOnProperty → MonitorAutoConfiguration 활성화 여부 결정
   @Value("${edgesquare.mode:monitor}") → WemDataSourceConfig, EdgeCoreService, StartManagerSpringApplication
  ↓
③ LicenseService.postConstruct() (빈 생성 후)
   라이선스 로드 + em_preference 테이블에 edgesquare.mode 동기화
  ↓
④ 프론트엔드
   cm.getSystemProperty("edgesquare.mode") → DB 값 사용
   monitor이면 rt_device_statistics_onoff.xml 첫 페이지
```

## 미완료 — 이어서 작업 필요

### 라이선스 제약 API 추가 (sMaxAdmin, sExpiredDt)

#### 1) 관리자 수 제약 (`sMaxAdmin`)
- **방식**: 별도 API `GET /api/license/admin-status` 신규 생성 (기존 코드 수정 없음)
- **응답 설계**:
  ```json
  {
    "status": "OK | WARNING | BLOCKED",
    "current": 8,
    "max": 10,
    "message": "관리자 계정이 90%에 도달했습니다."
  }
  ```
- **프론트 흐름**: 관리자 등록 버튼 클릭 → admin-status API 호출 → OK(바로 등록) / WARNING(경고 팝업 후 등록) / BLOCKED(차단 팝업)
- **기준**: 90% 이상 WARNING, 100% 이상 BLOCKED, -1은 무제한

#### 2) 만료일 제약 (`sExpiredDt`)
- 적용 시점/방식 미결정 — 다음 세션에서 논의 필요

#### 3) 에이전트 수 제약 (`sMaxUser`)
- 이미 EdgeRegisterController에 구현됨 (토큰 빈 문자열 처리)
- 추가 작업 불필요
