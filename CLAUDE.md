# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.
**이 파일은 Claude Code가 이 리포지토리에 진입하면 자동으로 읽는 유일한 부트스트랩 파일이다.**

## 세션 시작 프로토콜 (매 세션 필수)

1. `daily/INDEX.md` 읽기 → 팀 구성, 주요 설정, 과거 작업 맥락 복원
2. `memory/session_state.md` 읽기 → 비어있지 않으면 **이전 작업 복원 모드** 실행
3. 오늘 날짜의 데일리 파일 존재 여부 확인 (`daily/YYYY-MM-DD.md`)
4. 없으면 새 데일리 파일 생성 (전날 미완료 작업 이관)
5. 있으면 이어서 작업 기록
6. `dev/active/`에 진행 중인 작업 문서가 있으면 읽고 이어가기
7. 역할: **팀장(team-lead)**으로서 10명 서브에이전트 운영 → `TEAM.md` 참조

### 이전 작업 복원 모드

`session_state.md`가 비어있지 않을 때 자동 실행:

1. 저장된 작업 상태를 **요약하여 사용자에게 보여준다**
2. 다음 선택지를 제시한다:
   - **[A] 이어서 작업** — 저장된 다음 단계부터 바로 진행
   - **[B] 작업 목록 보기** — 미완료 작업 전체 목록을 보여주고 선택
   - **[C] 새로운 작업** — 이전 작업은 보류하고 새 요청 대기
3. 사용자가 선택하면 그에 따라 진행
4. 복원 완료 후 `session_state.md` 내용을 초기화

## 세션 종료 / 날짜 변경 시

1. 당일 데일리 파일에 수행 작업/의사결정/미완료 항목을 **상세하게** 기록
2. `daily/INDEX.md`에 해당일 **핵심 키워드** 요약 추가/갱신
3. `dev/active/` 작업 문서 갱신
4. `git add . && git commit && git push` 실행

### 데일리 로그 작성 규칙

- **상세 기록**: 작업 배경/결정 이유/구체적 결과물/참고 자료를 상세히 기록
- **URL 원본 보존**: 사용자가 공유한 URL은 손실 없이 그대로 기록
- **1000줄 제한**: 파일 1개당 1000줄 초과 금지
- **분할**: 초과 시 `YYYY-MM-DD_1.md`, `_2.md` 순으로 넘버링, INDEX.md에도 반영

## 사용자 선호

- **모든 답변 끝에 컨텍스트 잔여량 표시** (예: `컨텍스트 잔여량: ~60%`)
- git: 자동 커밋/푸시 선호 (요청 시 즉시 실행)
- 응답 시 `[역할명]` 태그를 붙여 누가 답변하는지 명확히 표시
- 언어: 한국어 기본, 코드 주석은 영어 허용

## 컨텍스트 관리 전략

- 대화가 길어지면 핵심 내용을 데일리 파일에 기록 후 대화 압축 가능
- 과거 맥락 필요 시: `daily/INDEX.md`(요약) → 해당 날짜 md(상세) 순서로 참조
- 장기 작업: `dev/active/` 3파일 패턴(plan/context/tasks)으로 관리

### 세션/컨텍스트 연속성 프로토콜 (필수)

**모든 세션 종료(`exit`) 또는 컨텍스트 초기화(`/clear`) 전에 반드시 실행한다.**
사용자가 "재시작", "exit", "/clear", "나갔다 올게" 등의 의사를 표현하면 즉시 아래를 실행:

1. **`memory/session_state.md`에 현재 작업 상태 기록**
   - 진행 중인 작업 내용 및 진행도
   - 마지막 사용자 요청 (원문 그대로)
   - 다음에 이어서 해야 할 것 (구체적 단계)
   - 주요 의사결정 사항
   - 변경한 파일 목록 및 변경 내용 요약
   - 미해결 이슈/블로커
2. **당일 데일리 파일 갱신**: 수행 작업을 상세히 기록
3. **`daily/INDEX.md`**: 해당일 키워드 요약 갱신
4. **git 커밋/푸시**: 모든 변경사항 저장

> **복원 (세션/컨텍스트 시작 시)**: `memory/session_state.md`가 비어있지 않으면 읽고 자동 복원. 복원 완료 후 파일 내용을 초기화.

### 컨텍스트 부족 시 (~10% 이하)

위 연속성 프로토콜을 실행한 뒤, 사용자에게 안내:
"컨텍스트가 부족합니다. 상태를 저장했습니다. `/clear` 후 아무 메시지나 보내시면 이어갑니다."

> **주의**: `/clear`는 사용자만 실행 가능. Claude가 직접 실행할 수 없으므로 반드시 사용자에게 요청한다.

## 기술 스택

- **Backend:** Java 17+, Spring Boot 3.x, Spring Security, Spring Data JPA
- **Frontend:** HTML/CSS/JS, Thymeleaf/React/Vue (프로젝트에 따라)
- **Build:** Gradle (wrapper 사용: `./gradlew`)
- **DB:** PostgreSQL (주), MySQL, Redis (캐시)
- **Infra:** Docker, GitHub Actions, Nginx
- **Test:** JUnit 5, Mockito, RestAssured
- **API 문서:** Springdoc OpenAPI (Swagger UI)
- **코드 품질:** SonarQube, Lombok, Checkstyle

## 개발 워크플로우

새 기능 개발 시 즉시 코드를 작성하지 않는다. 다음 순서를 따른다:

1. **요구사항 상세화** — 무엇을 만드는지 문서화. 사용자 검토 후 진행.
2. **설계 문서** — 어떻게 만드는지 설계. 사용자 검토 후 진행.
3. **테스트 계획** — 어떻게 검증하는지 정의. 사용자 검토 후 진행.
4. **구현** — 문서에 따라 코드 작성.

예외: 명백한 1~2줄 버그 수정, 기존 패턴의 단순 반복, 사용자가 명시적으로 "바로 구현해줘"라고 요청한 경우.

## 산출물 관리 (output/)

작업 산출물은 **`output/` 디렉토리**에 저장한다.

| 하위 폴더 | 용도 |
|-----------|------|
| `output/dev/` | 개발 소스 관련 (QA 테스트 보고서, 코드 리뷰, 빌드 결과 등) |
| `output/docs/` | 문서 산출물 (기획서, 분석 보고서, 설계서 등) |
| `output/{기타}/` | 필요 시 용도별 하위 폴더 생성 |

- 파일명 규칙: `YYYY-MM-DD_{산출물명}.{확장자}`

## 자동화 설정 (.claude/)

### 컨텍스트 리셋 대응 — 3파일 패턴

장기 작업 시 `dev/active/`에 다음 3파일을 유지한다:
- `{task}-plan.md` — 전략 (왜)
- `{task}-context.md` — 결정사항 (무엇)
- `{task}-tasks.md` — 체크리스트 (어디까지)

세션 종료 전 갱신. 다음 세션에서 이 파일들로 복원.

## 리포지토리 파일 맵

```
claude-team/
├── CLAUDE.md              ★ [자동] 이 파일 — 부트스트랩
├── TEAM.md                ★ [자동참조] 10명 서브에이전트 정의
├── daily/
│   ├── INDEX.md           ★ [세션시작] 과거 맥락 복원 인덱스
│   └── YYYY-MM-DD.md        [필요시] 날짜별 상세 로그
├── team/                  ★ [자동] 페르소나별 작업 산출물 폴더
│   └── {persona-id}/       각 페르소나 개인 폴더 (10명)
├── output/                ★ 산출물 저장소
│   ├── dev/                 개발 소스 관련 산출물
│   └── docs/                문서 산출물
├── dev/
│   └── active/            ★ [세션시작] 진행 중 작업 문서
├── memory/
│   └── session_state.md   ★ 세션 연속성 상태 저장
└── .claude/
    ├── commands/            슬래시 커맨드
    ├── hooks/               훅 스크립트
    └── skills/              스킬 정의
```

**★ 표시 = Claude Code가 자동으로 인식하거나, 세션 시작 프로토콜에 의해 자동 로드**
