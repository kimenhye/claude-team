## 4. 라이선스 파일 구조

### 4.1 파일 정보

| 항목 | 설명 |
|------|------|
| 파일명 | `license_EdgeSquare` |
| 형식 | Base64 + DES 암호화 JSON |
| 위치 | `${application.path.home}/license/` |

### 4.2 라이선스 JSON 필드 매핑

| 발급 시스템 필드 | 라이선스 JSON 필드 | 설명 |
|-----------------|-------------------|------|
| MAX User | sMaxUser | 최대 Agent 수 (-1=무제한) |
| 관리자 수 | sMaxAdmin | 최대 관리자 수 (-1=무제한) |
| 서버 수 | sMaxServer | 최대 클러스터 서버 수 (-1=무제한) |
| (제품종류 기반) | sLicenseTrgtCd | 제품 코드 (32,33,34) |
| 만료일 | sExpiredDt | yyyyMMdd 형식 |

### 4.3 라이선스 JSON 예시

```json
{
  "sMaxAdmin": "10",
  "sMaxUser": "100",
  "sMaxServer": "3",
  "sLicenseTrgtCd": "32,33",
  "sExpiredDt": "20261231"
}
```

## 5. 제품 코드 (sLicenseTrgtCd)

| 코드 | 제품명 | 설명 |
|------|--------|------|
| 32 | Manager | Edge 디바이스 관리 (배포, 공지, 정책 등) |
| 33 | Monitor | 모니터링/로깅 (대시보드, APM, 로그 수집) |
| 34 | Deploy | 배포 전용 |
| 0 | - | 미등록 |

### 5.1 제품 조합별 동작 모드

| 조합 | edgesquare.mode | 활성화 모듈 |
|------|----------------|-------------|
| 32 | manager | Edge 관리 기능만 |
| 33 | monitor | 모니터링 기능만 |
| 32,33 | monitor | Edge 관리 + 모니터링 (풀 패키지) |
| 32,34 | manager | Edge 관리 + 배포 |
| 32,33,34 | monitor | 전체 기능 |

> sLicenseTrgtCd에 "33" 포함 시 monitor 모드, 미포함 시 manager 모드

## 6. 수량 제한 기준

### 6.1 MAX User (sMaxUser) - Agent/Edge 수

| 값 | 설명 | 시스템 동작 |
|------|------|-------------|
| 양수 (예: 100) | 최대 100대 접속 | 초과 시 신규 접속 차단 |
| -1 (제한없음) | 무제한 | 제한 없음 |

### 6.2 관리자 수 (sMaxAdmin)

| 값 | 설명 | 시스템 동작 |
|------|------|-------------|
| 양수 (예: 10) | 최대 10명 | 초과 시 경고 표시 (차단 없음) |
| -1 (제한없음) | 무제한 | 제한 없음 |

### 6.3 서버 수 (sMaxServer)

| 값 | 설명 | 시스템 동작 |
|------|------|-------------|
| 양수 (예: 3) | 클러스터 3대 | 정보성 표시 |
| -1 (제한없음) | 무제한 | 제한 없음 |

## 7. Feature Flag (기능 라이선스)

Feature Flag는 라이선스 파일과 별도로 `em_preference` 테이블에서 관리된다.

| Flag | 기본값 | 설명 | 영향 범위 |
|------|--------|------|-----------|
| license.feature.organization | true | 조직/부서 기능 | 공지/배포 대상 선택 |
| license.feature.group | true | 그룹 기능 | 공지/배포 대상 선택 |
| license.feature.distribute.apply | false | 배포 승인 워크플로우 | 파일 배포 승인 절차 |
| license.feature.mobileAppDist | true | 모바일 앱 배포 | 모바일 앱 배포 메뉴 |

### 7.1 Feature Flag 비활성 시 동작

| Flag 비활성 | UI 동작 |
|------------|---------|
| organization + group 모두 false | 전체 발송만 가능 |
| organization만 false | 조직 선택 제거, 그룹만 사용 |
| group만 false | 그룹 선택 제거, 조직만 사용 |
| distribute.apply = false | 승인 없이 직접 배포 |
| mobileAppDist = false | 모바일 앱 배포 메뉴 비활성화 |

## 10. 발급 절차

```
1. 고객 요구사항 확인
   - 고객사/프로젝트 정보
   - 제품 범위 (Manager/Monitor/Deploy)
   - 배포 환경 (On-Premise / Cloud K8S)
   - 서버 정보 (호스트명, IP, CPU)
   - 수량 제한 (Agent수, 관리자수, 서버수)
   - 라이선스 타입 (실버전/데모)

2. 기술지원시스템에서 발급
   - 고객등록 (미등록 시)
   - 라이센스 발급 화면에서 정보 입력
   - 라이선스 키 생성

3. 라이선스 파일 배포
   - ${application.path.home}/license/에 license_EdgeSquare 배치
   - 서버 재시작

4. Feature Flag 설정 (필요 시)
   - em_preference 테이블에 license.feature.* 값 설정
   - (라이선스 파일과 별도 설정)

5. 검증
   - /api/license/info API로 라이선스 정보 확인
   - UI에서 기능 활성화 상태 확인
```

## 11. 주의사항

| 서버 재시작 | 라이선스 파일 변경 후 반드시 서버 재시작 필요 |
