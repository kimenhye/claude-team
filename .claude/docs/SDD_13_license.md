## 1. 개요

### 1.1 목적
시스템 라이선스를 검증하고, 라이선스에 따라 기능을 활성화/비활성화하는 기능을 설계한다.

### 1.2 범위
- 라이선스 검증
- 기능별 라이선스 제어
- 라이선스 정보 조회

---

## 2. 기능 설계

| No | 기능명 | 설명 | 우선순위 |
|----|--------|------|----------|
| 1 | 라이선스 검증 | 라이선스 키 유효성 검증 | 상 |
| 2 | 기능 제어 | 라이선스별 기능 활성화/비활성화 | 상 |
| 3 | 라이선스 조회 | 현재 라이선스 정보 조회 | 중 |

## 4. 상세 설계

### 4.1 주요 서비스

| 서비스 | 설명 |
|--------|------|
| LicenseService | 라이선스 검증/관리 |

### 4.2 라이선스 기능 플래그

| 플래그 | 설명 | 영향 모듈 |
|--------|------|-----------|
| license.feature.organization | 조직 기능 사용 여부 | 조직 관리, 공지 |
| license.feature.group | 그룹 기능 사용 여부 | 그룹 관리, 공지 |

### 4.3 라이선스 적용 로직
- 프론트엔드: `cm.getSystemProperty()` 로 라이선스 플래그 조회
- 플래그에 따라 UI 컴포넌트 show/hide 처리
- 조직/그룹 모두 미사용 시 전체 발송만 가능
